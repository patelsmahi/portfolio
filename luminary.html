Luminary

Luminary is a navigation app built by App Team Carolina to help UNC Chapel Hill students find safe, accessible routes around campus. It allows users to plan paths that avoid obstacles like construction and uneven terrain, and includes a community feature for reporting new barriers in real time.

Problem: Luminary users had difficulty viewing the app comfortably in different lighting conditions, especially without a light mode option.

Solution & Outcome: In response to user feedback, I designed a light mode that improved visibility in bright environments while maintaining strong color contrast and accessibility standards. I also ensured the design stayed consistent with the app’s existing brand. This update made the app more flexible and comfortable to use without changing its core experience. The project reinforced how listening to users and making thoughtful visual adjustments can significantly improve usability and inclusivity.

Role: UI/UX Designer
Duration: 2 Months

Process

User Research & Insights

Feedback from early testing highlighted the need for a light mode. Users reported difficulty reading the dark interface in bright environments.Many requested customization options to suit personal preferences.

These insights shaped my approach to color choices, contrast ratios, and layout adjustments, ensuring the interface remained clear, consistent, and user-friendly.

Design Goals

The primary goals for the Luminary light mode redesign were to:

  • Enhance readability and comfort

    Ensure text, icons, and interface elements are easy to read in a bright environment.

  • Maintain brand consistency

    Adapt the existing dark mode style while keeping the app’s visual identity intact.

  • Support accessibility standards

    Ensure sufficient contrast and usability for all users, including those with visual impairments

  • Provide a seamless toggle experience

    Allow users to switch between dark and light mode without disrupting their workflow or navigation.

UI Iteration

With these goals in mind, I adapted Luminary’s existing interface into a light mode design. This process included testing contrast, readability, and visual hierarchy while preserving familiar navigation patterns.

Results

Working on Luminary gave me the opportunity to directly address user needs by introducing a light mode interface. While my role focused on adapting the existing design, the process pushed me to think critically about accessibility, consistency, and user preference. By maintaining brand identity and ensuring strong readability in both modes, I was able to expand the app’s flexibility without compromising its core design. This project reinforced the importance of listening to user feedback and demonstrated how even small interface adjustments can have a meaningful impact on overall usability
